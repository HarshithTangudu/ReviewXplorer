# Plan 01-01: Migrate Scrapers to Async Playwright

## Objective

Convert the `AmazonScraper` and `FlipkartScraper` logic to leverage full asynchronous browser automation using `playwright.async_api`, replacing the currently broken `requests` and `BeautifulSoup` system. This addresses aggressive anti-bot countermeasures (Requirement SCRP-01, SCRP-02).

## Requirements Addressed

- SCRP-01: Fix Amazon scraper extraction
- SCRP-02: Fix Flipkart scraper extraction
- SCRP-03: Handle blocked requests or captchas gracefully

## Tasks

<task>
<read_first>
- backend/scrapers/amazon.py
- backend/scrapers/flipkart.py
</read_first>
<action>
Rewrite logic in `amazon.py` to use `playwright.async_api`:
1. Remove `requests` and `BeautifulSoup` imports. Import `async_playwright` from `playwright.async_api` and `TimeoutError`.
2. In `scrape()`, run `async with async_playwright() as p:` to launch an asynchronous chromium browser with `headless=True`.
3. Create a browser context with a generic user agent like `"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/123.0.0.0 Safari/537.36"`.
4. Navigate identically through pages `1` to `max_pages`. Iterate securely and wait for `[data-hook='review-body'], .review-text-content, .review-text` selectors up to 15,000ms. Extract all text elements.
5. If `TimeoutError` occurs during `page.wait_for_selector()`, gracefully catch the exception, print an error about capthas/blocking to the console, and return whatever target `all_texts` list we have already successfully collected.
</action>
